Welcome to TYPE!

TYPE is a beautiful, bold Ghost blog theme. It features big visuals and enough flexibility to fit every need, every style.

We at QBKL thank you for the choice you have made by purchasing our product!

This is a not a WordPress theme and will not work if you try to use it as one.

1. Installation

Installing TYPE is a very simple and straight forward job. Please consider the following:

1.1. Requires Ghost

In order to make use of this theme, you need to have the Ghost blogging platform installed. It's a free platform built on Node.JS support. Check out the Ghost documentation.

1.2. Installing the 'TYPE' theme

Setting up the theme is a very easy job! Basically, all you have to do is to move (ONLY) the contents of the _theme folder to your server, inside your Ghost installation, inside the ghost/content/themes/ folder.

We suggest customizing the theme settings and design before uploading.

You can find more quality information about Ghost on their own site at Ghost Support and User Guide.

2. Customization

Customizing TYPE is not really rocket science. Let's have a look at things:

2.1. Color themes

Type comes pre-packed with 9 color schemes. One default theme, 4 additional color/gradient schemes based on the seasons of the year and 4 monochrome themes in dark or light shades.

Open the default.hbs file. Simply adding a class to the BODY tag like so: <body class="{{body_class}} theme-summer"> tag will change the color scheme. Do not remove the {{body_class}} helper. It is very important to the theme's styling. You don't need to add anything for the default theme.

Here is a list of all the available color theme classes:

  • theme-spring
  • theme-summer
  • theme-autumn
  • theme-winter
  • theme-mono-light
  • theme-mono-dark
  • theme-white
  • theme-black

If you decide to go for the theme-mono-light or theme-white, you'll also need to switch to dark fonts for better contrast in all the theme's heroes. You'll find all of them in the partials/heroes folder. Open them and replace every instance of light-font with dark-font.

2.2. Custom logo

Adding your own blog logo is done simply by editing the settings of your Ghost installation under the Setting/General tab.

2.3. Adjusting the author information

Adjusting the author image, cover photo and information is done simply by editing the settings of your Ghost installation under the Setting/User tab.

We only display the author's description, website - if available and cover photo in the author's archive page's hero. On posts, only the author's name is displayed and linked to their archive.

2.4. Main navigation

The theme's main navigation is located in the partials/page-header.hbs partial. Locate the following and adjust to your needs:

<nav id="nav">
	<ul class="clearfix">
		<li><a href="{{@blog.url}}">Home</a></li>
		<li><a href="/about-type/">About</a></li>
		<li><a href="#">Purchase</a></li>
	</ul>
</nav>
Do not edit the mobile nav elements. The mobile navigation will be generated automatically based on the main navigation.

2.5. Cover images

Every cover image that you add, be it blog cover image, post cover image or author cover image, it will be used in the blog's home hero, post's hero or author's hero.

2.6. Archive types

For more flexibility, we've included 4 display types for your posts' archives:

  • Single column, full width list - posts-list.hbs
  • Single column list with sidebar - posts-sidebar.hbs
  • Two-columns grid - posts-duogrid.hbs
  • Three-columns grid - posts-triogrid.hbs

All 4 archive type files can be found in the partials/archives folder. These files are loaded into the main archive type templates of the theme:

  • home.hbs
  • index.hbs
  • tag.hbs
  • author.hbs

The way the theme has been split in such partial files allows you to set up different display styles for homepage, general archive, tag and authors and thus offers more flexibility to the theme.

2.7. Disqus comments

Disqus comments are displayed at the bottom of every post. The embeded Disqus code can be found in the partials/post-comments.hbs file.

In order to setup Disqus, you'll need to open the assets/js/custom.scripts.js file and locate the following lines at the beginning of the file:

var disqusComments = true;			// Set to false to hide comments
var disqusUser = 'yourdisqus_shortname';	// Set to your own Disqus Shortname

Replace yourdisqus_shortname with the shortname provided by Disqus.com after you've registered and added your website. Set the disqusComments variable to false if you wish to disable the comments.

2.8. Sidebar

The theme include a sidebar template - sidebar.hbs - which in turn loads the currently avaialble widget partials from partials/widgets:

  • About - widget-about.hbs
  • Must read list - widget-must-read.hbs
  • Sidebar menu - widget-side-menu.hbs
  • Dribbble shots - widget-dribbble.hbs

Width the exception of the Dribbble widget, these "widgets" are currently HTML based and thus, they require manual editing. As soon as Ghost will offer a way to manage the widgets dynamically from the dashboard, we will implement such management.

Let's have a look at how the widgets are loaded:

<div id="sidebar" class="col-sm-4 col-lg-4 col-lg-offset-1" role="complementary">

	{{> widgets/widget-about}}

	{{> widgets/widget-must-read}}

	{{> widgets/widget-side-menu}}

	{{> widgets/widget-dribbble}}

</div>

If you'd like to stop loading one of the widgets, you cad obviously delete the line loading it or simply comment it with a simple ! sign, like so:

{{!> widgets/widget-must-read}}
2.8.1. Dribbble widget

The Dribbble shots widget is a fully automated widget that pulls the latest shosts based on a few variables that you need to setup in the assets/js/custom.scrips.js file, at the beginning:

var dribbbleWidget = true;	// Set to false to hide widget
var dribbbleUser = 'Username';	// Set to your own Dribbble username
var dribbbleMaxShots = 6;	// Adjust the number of shots display. Even numbers.

3. Content goodies

We've included in the theme some special styling that allows you to create beautiful quotes, add floated images and more elements. Let's have a look at these elements:

3.1. Photos

Besides the regular way to include photos in Ghost, we've included a way to float images to the left, right or make the really wide, extending beyond the size of the container. These effects are best visible in the no-sidebar post and page displays.

<div class="pull-left"><img src="image_url_here.jpg" alt=""></div>

This pulls the image to the left.

<div class="pull-right"><img src="image_url_here.jpg" alt=""></div>

This pulls the image to the right.

<div class="wide"><img src="image_url_here.jpg" alt=""></div>

This forces a really wide image.

Add the mono class to force images to load in black and white.

To get the photo image URL, load the image as you normally would via Ghost's editor, then copy its URL for the auto-generated markdown.

![Alt image title here](image_url_here.jpg)

3.2. Quotes

Similar to floated images, you can add left or right pulled quotes. The default quote style, the one added via Ghost's markdown, is wide.

<blockquote class="pull-left">Quote text here</blockquote>
<blockquote class="pull-right">Quote text here</blockquote>

The above examples will create a left pulled quoted (first) or a right pulled quote (second).

3.3. Buttons

Adding a button means creating an HTML link with the added class button:

<a href="#" class="button">Button</a>
